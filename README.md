# Gmail 모듈 속성 상세 안내 (Watch Emails)

Make.com에서 Gmail의 `Watch Emails` (이메일 수집/감시 트리거) 모듈을 설정할 때 나타나는 주요 속성들에 대한 설명입니다.

| 속성명 (Property) | 필수 여부 | 실제 역할 설명 | 권장 설정 방식 |
| :--- | :---: | :--- | :--- |
| **Connection** | **필수** | Make가 어떤 구글 메일함을 뒤질지 계정을 연결하고 인증하는 곳입니다. | 메일을 읽어올 나의 구글 계정을 연동합니다. |
| **Filter type** | **필수** | 메일을 걸러내는 전체적인 필터 규칙 방식을 선택합니다. | 가장 직관적이고 다루기 쉬운 **`Simple filter`**를 선택합니다. |
| **Folder** | **필수** | 내 메일함 내에서 어떤 폴더(편지함)를 집중적으로 감시할지 정합니다. | 구글 알리미 등 새 메일이 들어오는 기본 위치인 **`Inbox`**(받은편지함)를 선택합니다. |
| **Label** | 선택 | 특정 라벨(태그)이 지정되어 붙은 메일만 솎아내고 싶을 때 사용합니다. | 별도의 특수한 분류 기준이 없다면 **비워둡니다(Leave empty).** |
| **Criteria** | **필수** | 메일함에 있는 메일 중 어떤 상태(읽음/안 읽음 등)의 메일을 가져올지 정하는 핵심 기준입니다. | 새로 도착한 뉴스 메일만 수집하기 위해 **`Only unread emails`**(읽지 않은 메일만)를 선택합니다. |
| **Mark email message(s) as read when fetched** | 선택 | Make가 메일 데이터를 성공적으로 긁어가고 나면, 실제 Gmail 앱 내에서도 해당 메일을 '읽음' 상태로 강제 전환할지 결정합니다. | 메일함에 읽지 않은 메일 숫자가 무한정 쌓이는 것을 막기 위해 **`Yes`** 설정을 추천합니다. |
| **Limit** | **필수** | 한 번 자동화 주기가 돌아갈 때 최대 몇 개의 메일을 한꺼번에 긁어올지 개수를 제한합니다. | 폭탄 수집으로 인해 무료 플랜 횟수가 급격히 차감되는 것을 방지하기 위해 **`2` 또는 `3`** 정도의 작은 숫자를 적어줍니다. |
